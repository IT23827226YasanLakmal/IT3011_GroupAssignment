## Project Overview

This repository contains all data, R scripts, outputs, and documentation for our IT3011 group assignment. The study investigates student self-assessment behaviours using statistical methods covering descriptive analysis, hypothesis testing, and predictive modelling.

---

## Repository Structure

```text

    raw/                    # Original, unaltered data (e.g., survey_results.csv)

    01_data_cleaning.R      # Handling missing values and outliers (LO1, LO4)
```

---

## How to Run

1. Open `IT3011_Project.Rproj` in **RStudio** - this sets the working directory automatically.
2. Run scripts **in order**:
   - `01_data_cleaning.R` - loads raw data, handles missing values and outliers
   - `02_descriptive.R` - generates summary statistics and visualisations
   - `03_inferential.R` - runs hypothesis tests
   - `04_predictive.R` - builds regression or time series models
3. All outputs (plots, tables) are saved to the `output/` folder.

## Key Workflow Notes

### .gitignore
Ensures temporary R files (.Rhistory, .RData) and any sensitive raw data are **not pushed** to the shared repository, preventing clutter and privacy issues.

### Script Numbering (01_, 02_, ...)
Numbered scripts make the **logical flow of analysis** immediately clear to every group member and the module assessor.

### AI Declaration Folder
Every use of AI assistance must be logged in `docs/AI_declarations/`. Record:
- The **prompt** used
- The **purpose** of the AI assistance
- The **date** it was used

Do this immediately after each AI interaction - do not leave it until Week 6.

### Git Best Practices
- **Pull before you push** - always fetch the latest changes before committing.
- **Commit often** with clear messages, e.g. `Add outlier removal to 01_data_cleaning.R`.
- **One script per analytical phase** - avoid editing multiple scripts in a single session to minimise merge conflicts.
